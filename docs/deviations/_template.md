# Governance Deviation Log

> This file records instances where governance rules were bypassed or overridden. Each entry is logged by the AI agent that detected the deviation. This log exists for auditability — it does not block work, but it ensures that all deviations are visible and traceable.

---

## Entry Format

Copy the block below for each deviation:

```md
### DEV-YYYYMMDD-NNN

- **Date:** YYYY-MM-DD
- **Developer:** [Name or "unspecified"]
- **AI Agent:** [Claude Code / GitHub Copilot / Cursor / Other]
- **Rule Deviated From:** [Rule number and name, e.g., "Rule 1: Respect the Hierarchy"]
- **What Was Skipped:** [Specific governance step that was bypassed]
- **Justification Given:** [Developer's stated reason, or "None provided"]
- **Risk Assessment:** [Brief assessment of what this deviation risks — e.g., "Feature implemented without spec; requirements may be incomplete or misunderstood"]
- **Resolution:** [How the deviation was resolved, if applicable — e.g., "Spec created retroactively", "Accepted as-is per developer override"]
```
